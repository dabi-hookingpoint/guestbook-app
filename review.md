# PRT (Peer Review Template) 리뷰 결과

## ☑️ 1. 주어진 문제를 해결하는 완성된 코드가 제출되었나요? (완성도)

**충족** — 근거:
- Supabase를 백엔드로 쓰는 정적 방명록 앱으로, 등록/조회 기본 기능에 더해 **삭제 기능**까지 구현되어 튜토리얼 기본 요구를 초과 달성 (`app.js`의 `deleteEntry`, `schema.sql`의 delete policy)
- 배포 링크에서 직접 테스트: 이름+메시지 입력 → 저장 → 새로고침해도 유지 → 삭제 버튼 클릭 시 정상 삭제까지 전부 실제 DB 기준으로 확인. 콘솔 에러 없음
- `escapeHtml()`로 XSS 이스케이프 처리, `schema.sql`에 RLS 정책과 `check` 제약(글자 수 제한)까지 반영되어 있어 꼼꼼함
- README의 "보안 체크" 섹션도 anon key 노출이 왜 안전한지 명확히 설명

**발견한 흠 (사소하지만 실재함):**
- index.html:134 및 배포된 페이지의 "GitHub 저장소" 링크가 본인 저장소(`dabi-hookingpoint/guestbook-app`)가 아니라 `jihoon2409-code/guestbook-app`(원본/다른 사용자 저장소로 추정)를 가리키고 있음. 포크 후 링크를 자기 저장소로 안 바꾼 것으로 보임 — 리뷰어가 코드를 확인하려면 엉뚱한 곳으로 가게 됨. (→ `fix/github-link-in-index` 브랜치에 수정 push 완료, PR 생성 대기 중)

## ☐ 2. 프로젝트에 대한 회고가 상세히 기록 되어 있나요? (회고, 정리)

**미충족** — README.md 확인 결과:
- **트러블슈팅/새로운 시도**: `<!-- TODO: ... -->` 플레이스홀더만 남아있고 미작성
- **회고**: "배운 점:", "아쉬운 점:", "느낀 점:" 라벨만 있고 내용이 전부 비어있음
- "어려웠던 점" 항목은 아예 존재하지 않음 (템플릿에서 요구하는 4개 항목 중 3개만 라벨 존재, 내용은 0개)

→ 코드 완성도는 높지만, 회고 파트는 사실상 작성되지 않은 상태.

- **dabi 본인이 작성한 커밋은 `42e2678` 하나뿐.** 커밋 메시지는 "회고/트러블슈팅 섹션 추가"라고 되어 있지만, 실제로는 코드 주석(핵심 로직 설명)만 추가되었고 회고/트러블슈팅 섹션은 제목만 만들어졌을 뿐 내용은 비어있음 — 항목 2의 미충족 판정과 일치.
